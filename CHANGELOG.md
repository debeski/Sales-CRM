# Changelog

## v0.1.2
- **Fix — sidebar highlighted "Invoices" on every sales page**: The invoice list was mounted at the app root `/sales/`, so dlux's prefix-based sidebar active check (`path.startswith(item_url)`) lit up Invoices on `/sales/dashboard/`, `/sales/customers/`, etc. Moved it to `/sales/invoices/` (name unchanged, all `{% url %}` refs auto-update) and redirected bare `/sales/` → dashboard. Now each sales page highlights only its own sidebar entry.
- **Cash-deposit batch combobox on payments**: The invoice payment form's "Cash Deposit" dropdown is now a `<datalist>` search-and-add combobox (same pattern as the customer field): type a batch reference to link an existing deposit or create one on the fly (`PaymentCreateView._sync_deposit`, find-or-create by `reference`, case-insensitive). A deposit that carries payments is a **batch** whose amount auto-sums its linked payments — `CashDeposit.recalc_amount()` runs from `Payment.save()/delete()` (reassigning a payment recomputes both the old and new batch), and `CashDepositForm` locks the amount field (display-only) while it has linked payments. The link stays fully optional (blank reference → no deposit). Deposits remain an optional cash-reconciliation tool for field/hand-over collection; everyday counter sales just record a Payment.
- **Fix — inline payments silently failed**: `PaymentForm` included the required `paid_at` field, but the invoice-detail quick-pay form only renders amount/method/deposit — so every UI payment failed validation ("paid_at is required") and redirected without saving (a `302`, no record). Removed `paid_at` from the form; it now falls back to its model default (`timezone.now`) on save. Earlier tests created `Payment` rows directly, bypassing the form, so they missed it — added `PaymentFormTests` covering the form path.
- **Translated permission labels in the group manager**: Added `perm_<codename>` AR/EN keys to `catalog`/`finance`/`sales` `translations.py` for every model permission (CRUD) plus the custom ones (`issue_invoice`, `cancel_invoice`, `view_sales_report`, `manage_exchangerate`, `confirm_cashdeposit`). dlux's `custom_permission_str` only localizes the permission *verb* (`can_view`→عرض) and leaves the model name in English, and the grouped-permission widget resolves each checkbox via `perm_<codename>` first — so without these keys the Arabic group manager showed e.g. "عرض Customer". Now it reads "عرض العملاء". English falls back to Django's default label unchanged.
- **Fix — `ProductTable` stock cell crash on Django 6.0**: `render_stock_qty` called `format_html('<span>—</span>')` with no placeholder args, which Django 6.0's stricter `format_html` rejects (`TypeError: args or kwargs must be provided`). Only non-stock-tracked products hit this branch, so it surfaced once demo data added such a product; now passes the dash as an arg. Added a `ProductTableRenderTests` regression test.
- **Demo Data Seeder**: New `sales/management/commands/seed_demo.py` (`python manage.py seed_demo`, `--reset` to rebuild) seeds a full test dataset — two `ExchangeRate`s (official + parallel), three `Category`s, six `Product`s (with barcodes, opening stock via `StockMovement`, a low-stock pair and an LYD-override "unrelated goods" item), five `Service`s, four `Customer`s, six `Invoice`s spanning every status (draft/issued/partial/paid/cancelled) with `Payment`s, and two `CashDeposit`s (pending + confirmed). Reference data is idempotent via `get_or_create`; invoices/deposits seed only when none exist (or after `--reset`). Issued invoices go through `issue_invoice()` so stock draws down and totals freeze realistically.
- **Form-Only Dynamic Modals**: Added project-scoped `scoped_modal_manager` / `scoped_modal_delete` routes (`config/urls.py`) that render dlux `DynamicModalManagerView` with `show_table=False`, so Add/Edit modals show only the form — no embedded records table. `common/views.ScopedListView` now emits `add_modal_url` + `modal_base_url` + `modal_delete_url`, and the new `common/static/common/js/scoped_crud.js` intercepts DluxTable `dlux:record:{view,edit,delete}` events (calling `preventDefault()` to bypass the dlux full-page fallback that pointed at non-existent `/{app}/{id}/edit/` routes) to open the form-only modal via `dlux:dynamic_modal:open` (or POST the delete). All modal `ModelForm`s set `refresh_parent = True` so the list reloads after a save.
- **Dashboard External Reference Rates (CBL + Black Market)**: `finance/services.py` scrapes two USD→LYD sources server-side (stdlib `urllib`; CSP forbids a browser cross-origin fetch): the **official** rate from `cbl.gov.ly` (`fetch_cbl_usd_rate`, cached `finance:cbl_official_usd_rate`) and the **black-market** rate + trend arrow from `eanlibya.com` (`fetch_ean_usd_rate`, cached `finance:ean_black_market_usd_rate`); both cached with **no expiry** — a value only ever changes when a later scrape succeeds, so a prolonged source outage never wipes the last-known rate. New `finance/tasks.refresh_market_rates` Celery task refreshes both (each independent) via `config/celery.py` `beat_schedule` (every 3h) plus a `worker_ready` warm-up on boot. Because `web` is network-isolated (`switch_pos_internal`, `internal: true`), the scraping runs in **celery** — which is now also attached to the egress bridge `switch_pos_net` — and the web tier reads the rates cache-only from Redis. The dashboard exchange-rate card shows the in-house **custom** rate beside the **official (CBL)** and **black-market (EAN)** rates, with the black-market trend (↑/↓), source dates, and the custom-vs-black-market gap.
- **Customer Search-and-Add Combobox**: The invoice header replaces the separate customer FK dropdown + name field with one `<datalist>`-backed text input (`InvoiceForm.customer` → `HiddenInput`). Typing/selecting matches an existing `Customer`, sets the hidden FK, and autofills phone/address; `_InvoiceEditorView._sync_customer()` binds or creates the `Customer` on save and backfills its blank phone/address, so every entered customer is saved and autofills on re-entry.
- **Filters via `advanced_filter_helper`**: `ScopedListView.get_filterset` now drives every list filter through dlux `advanced_filter_helper`, reading a per-`FilterSet` `advanced_config` (view-overridable via `filter_config`). Each config puts the keyword search + primary dropdown in the pill-shaped primary row and the rest inside the collapsible **"More filters"** advanced section (e.g. invoice date range, `is_active`, `unit`, `status`). The list templates render the filter with the **`{% crispy filter.form %}` tag** (not the `|crispy` filter, which silently ignores `form.helper` — the reason the advanced layout wasn't appearing). Added AR/EN strings `label_keyword`/`label_is_active`/`label_barcode`/`label_customer_name`, `search_placeholder`, `filter_advanced_search_action`, `filter_from`/`filter_to`, so every search box gets a translated placeholder and no filter renders as an unlabeled "unknown" field, with no dangling advanced toggle.
- **Translated dropdown options (`translate_choices`)**: New `common/forms.translate_choice_fields()` localizes every static-choice dropdown's *option* labels through dlux `translate_choices` (`choice_<value>` DLUX_STRINGS keys), applied to all list filters (in `get_filterset`) and to the modal `ModelForm`s (Product/Service/StockMovement/ExchangeRate/CashDeposit/Payment). Added the full `choice_*` AR/EN key set (status/method/movement/service-type/source/unit/kind/yes-no-any). This also fixed a pre-existing bug where django_filters `ChoiceField` dropdowns still showed `---------` as the first option: `set_first_choice` set the label on `field.choices`, but the widget renders from `widget.choices` — the helper now writes the translated, placeholder-preserving choices onto the widget, so filter dropdowns finally show their label (e.g. "Status"/"الحالة") as the first option.
- **Product Barcode Column**: Surfaced the existing `Product.barcode` field as a column in `ProductTable` (already searchable via `ProductFilter`).
- **Tests**: Added `finance.CblRateScraperTests` covering USD-row parsing (disambiguated from the Canadian/Australian dollar rows) and the network-failure fallback.

## v0.1.1
- **Tag-Driven Releases & GitHub Release Automation**: Replaced the branch-triggered `docker-publish.yml` (which pushed `debeski/sales:latest` on every `main` push but never created a GitHub Release) with the `app-composer`-style two-workflow setup. `release.yml` now fires on `v*` tags: it verifies `tag == VERSION`, runs `scripts/smoke-test.sh` against a freshly built image, pushes multi-arch (amd64+arm64) `debeski/sales:<ver>` + `:latest`, and publishes a **GitHub Release** from this `CHANGELOG.md` section via `softprops/action-gh-release`. A pushed git tag now becomes a real Release instead of only showing under Tags.
- **CI Gate**: `ci.yml` runs Django `check` + the test suite (`config.settings_dev_sqlite`) and a Docker build + runtime smoke test on every push/PR to `main`.
- **Version Bump**: `VERSION` → `0.1.1` (the release workflow fails the build if the pushed tag and `VERSION` disagree).

## v0.1.0
- **Modular App Architecture**: Scaffolded three DjangoLux apps via `dlux startapp --register` with a strict dependency layering `finance ← catalog ← sales`. Added a non-app `common` package (`common/views.py`, `templates/common/scoped_list.html`) holding the reusable `ScopedListView` so every list page wires `SingleTableMixin` + `FilterView` + `setup_filter_helper` + dynamic-modal "Add" in a few lines.
- **Finance — Global Exchange Rate**: `ExchangeRate` (append-only USD→LYD history; newest row is the live rate, cached under `finance:current_usd_lyd_rate`) with `manage_exchangerate` permission. Central `finance/services.py` (`get_current_rate`, `usd_to_lyd`, `lyd_to_usd`, `quantize_lyd`) is the single source of conversion math; ships a `DEFAULT_RATE` fallback surfaced as "no rate set".
- **Finance — Cash Deposits**: `CashDeposit` (ايداع نقدي) with `pending/confirmed/rejected` workflow, `confirm_cashdeposit` permission, and a `CashDepositReviewView` confirm/reject action.
- **Catalog — Hybrid Pricing**: `Product` priced in USD (`cost_usd` + `markup_percent` → `effective_price_usd`) auto-converted to LYD live, with an optional per-item `price_lyd_override` (the agreed hybrid model). `Service` offerings (installation/maintenance/warranty/delivery) priced in USD, LYD, or "per job". `Category` grouping. Auto-generated `sku` (`P00001`).
- **Catalog — Stock Ledger**: `StockMovement` (in/out/adjustment) is the authoritative inventory ledger; `Product.stock_qty` is excluded from the form and only changes through movements (applied atomically with `F()` on insert). Low-stock detection via `reorder_level`.
- **Sales — Invoices**: `Invoice` with auto number (`INV-000001`), `draft/issued/partial/paid/cancelled` lifecycle, and a **frozen** per-invoice `exchange_rate` plus per-line frozen `unit_price_lyd` so historical totals never shift when the rate moves. `InvoiceItem` supports product, service, and free-text custom lines with snapshot descriptions. `Customer` (optional; walk-ins allowed) and `Payment` (auto-maintains `amount_paid`/status, optional link to a `CashDeposit`).
- **Sales — Invoice Editor & Lifecycle**: Full-page multi-line editor (`InvoiceItemFormSet`) with a vanilla-JS add-row, live line/subtotal totals, and product/service price auto-fill from an embedded price map. `sales/services.py` `issue_invoice` (snapshots the rate record, draws down stock) and `cancel_invoice` (restores stock), both transactional. Detail page with issue/cancel/print/record-payment actions; standalone print-ready HTML invoice (`invoice_print.html`) branded from `APP_CONFIG`.
- **Sales — Dashboard**: Stat cards (sales today/month, outstanding balance, live rate), recent invoices, low-stock list, and pending-deposit alert. Intended as the system home page.
- **Roles**: `python manage.py seed_roles` creates a "Sales Staff" group (technicians/delivery reps) limited to selling, issuing invoices, taking payments, and recording cash deposits — no pricing, rate, or catalog access. Owner runs as a Django superuser.
- **Bilingual (AR/EN)**: `DLUX_STRINGS` label/model/app keys for all three apps so forms, filters and navigation render in Arabic or English via DjangoLux discovery.
- **Security & Audit**: Every domain model extends `ScopedModel` (audit trail, soft-delete, optional scope isolation) and every view is `LoginRequiredMixin` + `PermissionRequiredMixin` (`raise_exception=True`). Custom permissions: `manage_exchangerate`, `confirm_cashdeposit`, `issue_invoice`, `cancel_invoice`.
- **Sales Reporting + XLSX Export**: `sales/reports.py` (`build_sales_report`, `build_sales_report_xlsx`) aggregates issued/partial/paid invoices over a date window — total/collected/outstanding, by-status, daily, top products, top services, and an invoice list. `SalesReportView` (HTML, date-range filter) + `SalesReportExportView` (multi-sheet `.xlsx` via openpyxl, styled headers + money formats). Gated by a new `sales.view_sales_report` permission; the file-download route is marked `sidebar_exclude` so only the report *page* appears in navigation.
- **Negative-Stock Guard**: `issue_invoice` now aggregates demand per tracked product across lines and refuses to issue (transactional `ValidationError` naming each shortage) if any product would go negative — drafts and stock stay untouched on failure.
- **Dev tooling**: `config/settings_dev_sqlite.py` overlay for running migrations/checks locally without Postgres/Redis (dev only).
- **Launch**: Verified the DjangoLux Docker stack (`compose.yml` + `compose.dev.yml`) builds and starts via `./start.sh -d` (composer orchestrator, DEV mode, app on `http://localhost:90`).
- **CI/CD — tag-driven Docker Hub + GitHub Releases**: `.github/workflows/ci.yml` (push/PR to `main`: Django check + tests on `config.settings_dev_sqlite`, plus a Docker build + `scripts/smoke-test.sh` runtime gate) and `.github/workflows/release.yml` (on `v*` tag: verify `tag == VERSION`, smoke-test, multi-arch amd64+arm64 build & push `debeski/sales:<ver>` + `:latest`, then a GitHub Release from the matching `CHANGELOG.md` section via `softprops/action-gh-release`). `scripts/smoke-test.sh` boots the image and applies all migrations on in-image SQLite. Auth via `DOCKERHUB_USERNAME`/`DOCKERHUB_TOKEN` secrets; see `docs/RELEASING.md`. Patterned on the `app-composer` setup.
- **Full AR/EN Localization**: Promoted `common` to an installed (model-less) app hosting `common/translations.py` — shared `label_<col>` table-header keys (DjangoLux's django-tables2 patch localizes headers via these at render time), choice-label keys (`status_*`/`method_*`/`mtype_*`/`svctype_*`/`unit_*`/`source_*`/`kind_*`), and `ui_*`/`page_*`/`pr_*` chrome. Rewrote every hand-written template (invoice list/editor/detail/print, dashboard, report) to pull text from `DLUX_STRINGS`; `ScopedListView` now resolves page titles from `page_*_key`s; tables localize choice cells via a `common.i18n.t` helper and `Invoice.status_label`/`InvoiceItem.kind_label`/`Payment.method_label` properties; the printed invoice flips to `dir="rtl"` in Arabic. Previously only form/filter field labels were translated.
