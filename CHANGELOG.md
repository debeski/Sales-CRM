# Changelog

## v0.1.0
- **Modular App Architecture**: Scaffolded three DjangoLux apps via `dlux startapp --register` with a strict dependency layering `finance ← catalog ← sales`. Added a non-app `common` package (`common/views.py`, `templates/common/scoped_list.html`) holding the reusable `ScopedListView` so every list page wires `SingleTableMixin` + `FilterView` + `setup_filter_helper` + dynamic-modal "Add" in a few lines.
- **Finance — Global Exchange Rate**: `ExchangeRate` (append-only USD→LYD history; newest row is the live rate, cached under `finance:current_usd_lyd_rate`) with `manage_exchangerate` permission. Central `finance/services.py` (`get_current_rate`, `usd_to_lyd`, `lyd_to_usd`, `quantize_lyd`) is the single source of conversion math; ships a `DEFAULT_RATE` fallback surfaced as "no rate set".
- **Finance — Cash Deposits**: `CashDeposit` (ايداع نقدي) with `pending/confirmed/rejected` workflow, `confirm_cashdeposit` permission, and a `CashDepositReviewView` confirm/reject action.
- **Catalog — Hybrid Pricing**: `Product` priced in USD (`cost_usd` + `markup_percent` → `effective_price_usd`) auto-converted to LYD live, with an optional per-item `price_lyd_override` (the agreed hybrid model). `Service` offerings (installation/maintenance/warranty/delivery) priced in USD, LYD, or "per job". `Category` grouping. Auto-generated `sku` (`P00001`).
- **Catalog — Stock Ledger**: `StockMovement` (in/out/adjustment) is the authoritative inventory ledger; `Product.stock_qty` is excluded from the form and only changes through movements (applied atomically with `F()` on insert). Low-stock detection via `reorder_level`.
- **Sales — Invoices**: `Invoice` with auto number (`INV-000001`), `draft/issued/partial/paid/cancelled` lifecycle, and a **frozen** per-invoice `exchange_rate` plus per-line frozen `unit_price_lyd` so historical totals never shift when the rate moves. `InvoiceItem` supports product, service, and free-text custom lines with snapshot descriptions. `Customer` (optional; walk-ins allowed) and `Payment` (auto-maintains `amount_paid`/status, optional link to a `CashDeposit`).
- **Sales — Invoice Editor & Lifecycle**: Full-page multi-line editor (`InvoiceItemFormSet`) with a vanilla-JS add-row, live line/subtotal totals, and product/service price auto-fill from an embedded price map. `sales/services.py` `issue_invoice` (snapshots the rate record, draws down stock) and `cancel_invoice` (restores stock), both transactional. Detail page with issue/cancel/print/record-payment actions; standalone print-ready HTML invoice (`invoice_print.html`) branded from `APP_CONFIG`.
- **Sales — Dashboard**: Stat cards (sales today/month, outstanding balance, live rate), recent invoices, low-stock list, and pending-deposit alert. Intended as the system home page.
- **Roles**: `python manage.py seed_roles` creates a "Sales Staff" group (technicians/delivery reps) limited to selling, issuing invoices, taking payments, and recording cash deposits — no pricing, rate, or catalog access. Owner runs as a Django superuser.
- **Bilingual (AR/EN)**: `DLUX_STRINGS` label/model/app keys for all three apps so forms, filters and navigation render in Arabic or English via DjangoLux discovery.
- **Security & Audit**: Every domain model extends `ScopedModel` (audit trail, soft-delete, optional scope isolation) and every view is `LoginRequiredMixin` + `PermissionRequiredMixin` (`raise_exception=True`). Custom permissions: `manage_exchangerate`, `confirm_cashdeposit`, `issue_invoice`, `cancel_invoice`.
- **Sales Reporting + XLSX Export**: `sales/reports.py` (`build_sales_report`, `build_sales_report_xlsx`) aggregates issued/partial/paid invoices over a date window — total/collected/outstanding, by-status, daily, top products, top services, and an invoice list. `SalesReportView` (HTML, date-range filter) + `SalesReportExportView` (multi-sheet `.xlsx` via openpyxl, styled headers + money formats). Gated by a new `sales.view_sales_report` permission; the file-download route is marked `sidebar_exclude` so only the report *page* appears in navigation.
- **Negative-Stock Guard**: `issue_invoice` now aggregates demand per tracked product across lines and refuses to issue (transactional `ValidationError` naming each shortage) if any product would go negative — drafts and stock stay untouched on failure.
- **Dev tooling**: `config/settings_dev_sqlite.py` overlay for running migrations/checks locally without Postgres/Redis (dev only).
- **Launch**: Verified the DjangoLux Docker stack (`compose.yml` + `compose.dev.yml`) builds and starts via `./start.sh -d` (composer orchestrator, DEV mode, app on `http://localhost:90`).
- **CI — Docker Hub Publish**: `.github/workflows/docker-publish.yml` builds the repo-root `Dockerfile` with Buildx (gha layer cache) and pushes `debeski/sales:latest` + `debeski/sales:<git-sha>` to Docker Hub on push to `main`/`master` or manual dispatch. Authenticates via `DOCKERHUB_USERNAME` / `DOCKERHUB_TOKEN` repo secrets.
- **Full AR/EN Localization**: Promoted `common` to an installed (model-less) app hosting `common/translations.py` — shared `label_<col>` table-header keys (DjangoLux's django-tables2 patch localizes headers via these at render time), choice-label keys (`status_*`/`method_*`/`mtype_*`/`svctype_*`/`unit_*`/`source_*`/`kind_*`), and `ui_*`/`page_*`/`pr_*` chrome. Rewrote every hand-written template (invoice list/editor/detail/print, dashboard, report) to pull text from `DLUX_STRINGS`; `ScopedListView` now resolves page titles from `page_*_key`s; tables localize choice cells via a `common.i18n.t` helper and `Invoice.status_label`/`InvoiceItem.kind_label`/`Payment.method_label` properties; the printed invoice flips to `dir="rtl"` in Arabic. Previously only form/filter field labels were translated.
