# Switch Pos

This project was generated by `python -m dlux startproject` and already follows the DjangoLux baseline.

## First Run

1. Review and edit `.secrets/.env` before first boot.
2. Install your dependencies.
3. Run `python manage.py migrate`.
4. Run `python manage.py dlux_setup --skip-configure`.
5. Start the server and sign in as a superuser.
6. Complete `/sys/setup/`.

## Docker Notes

- `compose.yml` is the production-leaning baseline
- `compose.dev.yml` is the local dev override used by the generated composer launcher flow
- `./start.sh -d` is expected to target the dev override pattern, matching the `min_survey` setup
- nginx uses the official `nginx:latest` image and mounts `./.nginx/nginx.conf` into `/etc/nginx/conf.d/default.conf`
- the web service image defaults to a local tag via `WEB_IMAGE`, so it is not tied to a specific Docker Hub account
- Celery uses the same app image as `web` and starts with `python -m celery -A config worker -B --loglevel=info`
- keep the compose file in the standard inline-env pattern; do not wire `.secrets/.env` into services directly

## Generated Env File

The generated `.secrets/.env` includes only these bootstrap values:

- `DJANGO_SECRET_KEY`
- `POSTGRES_USER`
- `POSTGRES_PASSWORD`
- `PGADMIN_DEFAULT_EMAIL`
- `PGADMIN_DEFAULT_PASSWORD`
- `ADMIN_PASS`
- `DEFAULT_FROM_EMAIL`
- `SMTP_RELAY_HOST`
- `SMTP_RELAY_PORT`
- `SMTP_RELAY_USE_TLS`
- `SMTP_RELAY_USER`
- `SMTP_RELAY_PASSWORD`

The SMTP relay values are bootstrap/fallback values. For the normal UI-first path,
open System Setup step 3, choose delivery path `Internal SMTP relay`, choose
secret storage `Encrypted database secret`, and enter the upstream SMTP provider
settings there.

Keep `.secrets/.env` out of version control and replace all default passwords before deployment.

## App Workflow

- Create new apps with `python -m dlux startapp app_name`
- Use `python -m dlux startapp app_name --register` if you want DjangoLux to patch `INSTALLED_APPS` and project URLs for you
- Generated apps follow the discovery-friendly Dlux layout: model-adjacent forms, filters, tables, views, translations, templates, and tests
- Keep app-local translations in `translations.py`
- Keep forms, filters, and tables in their conventional files so DjangoLux discovery can find them automatically
